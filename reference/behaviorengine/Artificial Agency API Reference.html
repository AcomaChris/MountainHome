Authentication

All API requests should include your API key in an Authorization HTTP header, like this:

Authorization: Bearer API_KEY

Versioning

Requests should include an AA-API-Version indicating the API version that the client supports, like this:

AA-API-Version: 2025-05-15

    "message": "...", // a human-readable error message
    "trace": "..."    // a string that uniquely identifies the failed API request
  }
}

Agent Messages

Agent messages allow the client to send state updates to the agent. Which messages an agent accepts and how these messages are interpreted is determined by the agent's configuration.

The word 'message' is just a shorthand for 'Agent message'. Any time you see the word 'message' in this documentation, it refers to an Agent message.

All messages include these base fields:

{
  "message_type": "...", // the static string identifying the message type
  "channels": ["..."],   // the channels to send this message on
  "ignorable": false     // When false, must be accepted by a component for the request to be valid
}

Specific message types may include additional fields.

ContentMessage

A message that contains unstructured content, such as agent perceptions. Most components that accept this message will accumulate the content over time.

message_type
string (Message Type)
Default: "ContentMessage"
Value: "ContentMessage"
channels
Array of strings (Channels)
ignorable
boolean (Ignorable)
Default: false

When False, a request that includes this message will be considered invalid if no components accept the message. When True, the message will be silently dropped if no component accepts it.

content
required
string (Content)

An arbitrary string of content.

FunctionCall (object) or null

Details linking the content in this message to a previously-generated function.

{
  • "message_type": "ContentMessage",
  • "channels": [ ],
  • "ignorable": false,
  • "content": "You joined the server 0.7 seconds ago. Current UTC time in the real-world is: 07:05:10",
  • "function_call": {
    • "id": "string",
    • "type": "failure"
    }
}

KVMessage

A message that contains a key-value pair. When used with a store that supports hierarchical keys, the key can be a path to a nested value. Most components that accept this message will only track the most recent value for each key.

message_type
string (Message Type)
Default: "KVMessage"
Value: "KVMessage"
channels
Array of strings (Channels)
ignorable
boolean (Ignorable)
Default: false

When False, a request that includes this message will be considered invalid if no components accept the message. When True, the message will be silently dropped if no component accepts it.

key
required
string (Key)

The key of the key-value pair.

required
Value (string) or Array of Value (strings) or Value (null) (Value)

The value of the key-value pair. Setting the value to null will remove the key from the key-value store.

{
  • "message_type": "KVMessage",
  • "channels": [
    • "attributes"
    ],
  • "ignorable": false,
  • "key": "health",
  • "value": "5"
}

KVDelTreeMessage

A message that contains a key that should be deleted from a store that supports hierarchical keys. Components that accept this message should remove any values or sub-keys associated with the key, and clean up any nodes that are consequently empty.

message_type
required
string (Message Type)

The type of message

Value: "KVDelTreeMessage"
channels
Array of strings (Channels)
ignorable
boolean (Ignorable)
Default: false

When False, a request that includes this message will be considered invalid if no components accept the message. When True, the message will be silently dropped if no component accepts it.

key
required
string (Key)

The root key to delete. Any value at this key and any subkeys will be removed. This message will only remove a single key in KV stores that are not configured to use hierarchical keys.

{
  • "message_type": "KVDelTreeMessage",
  • "channels": [
    • "combat"
    ],
  • "ignorable": false,
  • "key": "character.status_effects"
}

PuppeteerMessage

A message to a puppeteer component that contains the ID of a specific puppet, and a list of messages related to that puppet. Puppeteer components know how to handle data for multiple puppets concurrently, but will silently ignore messages in the list that they do not accept.

message_type
string (Message Type)
Default: "PuppeteerMessage"
Value: "PuppeteerMessage"
channels
Array of strings (Channels)
ignorable
boolean (Ignorable)
Default: false

When False, a request that includes this message will be considered invalid if no components accept the message. When True, the message will be silently dropped if no component accepts it.

puppet_id
required
string (Puppet Id)

The ID of the puppet.

required
Array of ContentMessage (object) or KVMessage (object) or KVDelTreeMessage (object) (Contents)

Messages related to this puppet.

{
  • "message_type": "PuppeteerMessage",
  • "channels": [
    • "combat"
    ],
  • "ignorable": false,
  • "puppet_id": "lisa_135",
  • "contents": [
    • {
      • "content": "The goblin stabs you.",
      • "message_type": "ContentMessage"
      },
    • {
      • "key": "status.effect",
      • "message_type": "KVMessage",
      • "value": "bleeding"
      },
    • {
      • "content": "You take 3 damage.",
      • "message_type": "ContentMessage"
      },
    • {
      • "key": "status.health",
      • "message_type": "KVMessage",
      • "value": "5"
      }
    ]
}

Create Agent

Creates an agent within a session.

path Parameters
session_id
required
string
Request Body schema: application/json
required
object (UIConfig)

UI configuration for the agent. This is used to configure the presentation of the agent in the UI.

required
object (RoleConfig)

Define the core operating instructions for the agent.

required
object (PresentationConfig)

Configuration for presenting agent state to an inference service.

Array of any (Component Configs)

List of components to use for this agent. Components maintain state that is rendered into the LLM prompt.

Array of objects (Service Configs)

List of services to use for this agent. Services generate data to determine the agent's behavior.

agent_llm
required
string (Agent Llm)

The LLM to use for the agent. Must be the id of an LLM service defined in service_configs.

Create Session

Creates a session.

Request Body schema: application/json
required
project_id
required
string (Project Id)

ID of a project to attribute the new session to.

object (Metadata)
Default: {}

Set of key-value pairs to be attached to the new session. This can be useful for storing additional information about the object in a structured format.

Expires In (integer) or Expires In (null) (Expires In)
Default: 10800

A number of seconds that the new session should remain active for.

After this time has expired, requests to generate behavior within this session will fail. If null, this session will not be limited by time.

Max Requests (integer) or Max Requests (null) (Max Requests)

A number of requests that the new session should remain active for.

After this number of requests have been made, requests to generate behavior within this session will fail. If null, this session will not be limited by a number of requests.

Clone Session

Clones a session.

path Parameters
original_session_id
required
string
Request Body schema: application/json
required
Moment Id (string) or Moment Id (null) (Moment Id)

ID of the moment to clone all agents in this session at. Must be a moment within the original session. If null, clone all agents at the most recent moment in the session. Mutually exclusive with agent_moments.

Agent Moments (object) or Agent Moments (null) (Agent Moments)

A dictionary mapping agent IDs to the moment IDs they should be cloned at. Each moment must be within the original session. Only the agents included in this dictionary will be cloned; omitted agents will be dropped from the new session. If null, default to cloning all agents at the moment specified by moment_id. Mutually exclusive with moment_id.

Project Id (string) or Project Id (null) (Project Id)

ID of a project to attribute the new session to, or null to use the project ID associated with the original session.

object (Metadata)
Default: {}

Set of key-value pairs to be attached to the new session. This can be useful for storing additional information about the object in a structured format.

Expires In (integer) or Expires In (null) (Expires In)
Default: 10800

A number of seconds that the new session should remain active for. After this time has expired, requests to generate behavior within this session will fail. If null, this session will not be limited by time.

Max Requests (integer) or Max Requests (null) (Max Requests)

A number of requests that the new session should remain active for. After this number of requests have been made, requests to generate behavior within this session will fail. If null, this session will not be limited by a number of requests.
